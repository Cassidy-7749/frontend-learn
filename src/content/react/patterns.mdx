# 常見元件模式

React 的靈活性讓開發者可以用多種模式來組織元件。選擇正確的設計模式能大幅提升程式碼的可讀性、可維護性與可複用性。本篇將介紹前端開發中最常見且實用的 React 元件模式。

## Compound Components 模式

Compound Components（複合元件）是一種讓多個元件協同工作的模式，透過隱式共享狀態來提供直覺的 API。最經典的例子是 HTML 的 `<select>` 和 `<option>`。

```jsx
// 使用 Compound Components 的 API
<Tabs defaultValue="tab1">
  <Tabs.List>
    <Tabs.Trigger value="tab1">基本資訊</Tabs.Trigger>
    <Tabs.Trigger value="tab2">進階設定</Tabs.Trigger>
    <Tabs.Trigger value="tab3">帳號安全</Tabs.Trigger>
  </Tabs.List>
  <Tabs.Content value="tab1">
    <p>這裡是基本資訊內容</p>
  </Tabs.Content>
  <Tabs.Content value="tab2">
    <p>這裡是進階設定內容</p>
  </Tabs.Content>
  <Tabs.Content value="tab3">
    <p>這裡是帳號安全內容</p>
  </Tabs.Content>
</Tabs>
```

### 實作方式

```jsx
import { createContext, useContext, useState } from 'react';

// 建立共享 Context
const TabsContext = createContext(null);

function Tabs({ defaultValue, children }) {
  const [activeTab, setActiveTab] = useState(defaultValue);

  return (
    <TabsContext.Provider value={{ activeTab, setActiveTab }}>
      <div className="tabs">{children}</div>
    </TabsContext.Provider>
  );
}

function TabsList({ children }) {
  return <div className="tabs-list" role="tablist">{children}</div>;
}

function TabsTrigger({ value, children }) {
  const { activeTab, setActiveTab } = useContext(TabsContext);

  return (
    <button
      role="tab"
      aria-selected={activeTab === value}
      className={`tab-trigger ${activeTab === value ? 'active' : ''}`}
      onClick={() => setActiveTab(value)}
    >
      {children}
    </button>
  );
}

function TabsContent({ value, children }) {
  const { activeTab } = useContext(TabsContext);
  if (activeTab !== value) return null;

  return (
    <div role="tabpanel" className="tab-content">
      {children}
    </div>
  );
}

// 組合成命名空間 API
Tabs.List = TabsList;
Tabs.Trigger = TabsTrigger;
Tabs.Content = TabsContent;
```

<Callout type="tip" title="Compound Components 的優勢">
使用者不需要手動管理狀態與傳遞 props，元件內部透過 Context 自動協調。這提供了極佳的開發體驗（DX），同時保持了元件結構的靈活性。像 Radix UI、Headless UI 等知名元件庫都大量使用此模式。
</Callout>

## Render Props 模式

Render Props 透過將函式作為 props（或 children），讓父元件決定子元件的渲染方式：

```jsx
// Render Props 模式
function MouseTracker({ render }) {
  const [position, setPosition] = useState({ x: 0, y: 0 });

  useEffect(() => {
    const handleMove = (e) => setPosition({ x: e.clientX, y: e.clientY });
    window.addEventListener('mousemove', handleMove);
    return () => window.removeEventListener('mousemove', handleMove);
  }, []);

  return render(position);
}

// 使用方式
function App() {
  return (
    <MouseTracker
      render={({ x, y }) => (
        <div>
          <p>滑鼠位置：({x}, {y})</p>
          <div style={{
            position: 'absolute',
            left: x - 10,
            top: y - 10,
            width: 20,
            height: 20,
            borderRadius: '50%',
            background: 'red'
          }} />
        </div>
      )}
    />
  );
}
```

### 使用 children 作為函式

```jsx
function DataFetcher({ url, children }) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch(url)
      .then(res => res.json())
      .then(data => { setData(data); setLoading(false); })
      .catch(err => { setError(err); setLoading(false); });
  }, [url]);

  return children({ data, loading, error });
}

// 使用方式
<DataFetcher url="/api/users">
  {({ data, loading, error }) => {
    if (loading) return <p>載入中...</p>;
    if (error) return <p>錯誤：{error.message}</p>;
    return <UserList users={data} />;
  }}
</DataFetcher>
```

## Higher-Order Components（HOC）

HOC 是一個接受元件並回傳新元件的函式，用於共享橫切關注點（cross-cutting concerns）：

```jsx
// HOC：為元件注入認證邏輯
function withAuth(WrappedComponent) {
  return function AuthenticatedComponent(props) {
    const { user, loading } = useAuth();

    if (loading) return <LoadingSpinner />;
    if (!user) return <Navigate to="/login" />;

    return <WrappedComponent {...props} user={user} />;
  };
}

// 使用方式
const ProtectedDashboard = withAuth(Dashboard);
const ProtectedSettings = withAuth(Settings);
```

```jsx
// HOC：注入主題
function withTheme(WrappedComponent) {
  return function ThemedComponent(props) {
    const theme = useTheme();
    return <WrappedComponent {...props} theme={theme} />;
  };
}

// HOC 可以組合
const EnhancedComponent = withAuth(withTheme(withLogging(MyComponent)));
```

### HOC 的注意事項

```jsx
// ❌ 不要在 render 中建立 HOC（每次渲染都會建立新元件）
function App() {
  const Enhanced = withAuth(Dashboard); // 每次 render 都是新元件！
  return <Enhanced />;
}

// ✅ 在模組層級建立
const Enhanced = withAuth(Dashboard);
function App() {
  return <Enhanced />;
}
```

<Callout type="info" title="HOC 已逐漸被 Custom Hook 取代">
在 Hook 出現之前，HOC 和 Render Props 是 React 中複用邏輯的主要方式。如今，大多數場景都推薦使用 Custom Hook。不過理解 HOC 仍然重要，因為許多舊程式碼和部分場景（如 `React.memo`、`forwardRef`）仍使用此模式。
</Callout>

## Custom Hook 取代 HOC / Render Props

Custom Hook 提供了更簡潔、更直覺的邏輯複用方式：

```jsx
// 將 HOC 改寫為 Custom Hook
function useAuth() {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    checkAuth().then(u => {
      setUser(u);
      setLoading(false);
    });
  }, []);

  return { user, loading, isAuthenticated: !!user };
}

// 將 Render Props 改寫為 Custom Hook
function useMousePosition() {
  const [position, setPosition] = useState({ x: 0, y: 0 });

  useEffect(() => {
    const handleMove = (e) => setPosition({ x: e.clientX, y: e.clientY });
    window.addEventListener('mousemove', handleMove);
    return () => window.removeEventListener('mousemove', handleMove);
  }, []);

  return position;
}

// 使用起來非常直覺
function Dashboard() {
  const { user, loading } = useAuth();
  const { x, y } = useMousePosition();

  if (loading) return <Spinner />;
  if (!user) return <Navigate to="/login" />;

  return <div>歡迎 {user.name}，滑鼠位置：({x}, {y})</div>;
}
```

### 比較三種模式

| 特性 | HOC | Render Props | Custom Hook |
|------|-----|-------------|-------------|
| 邏輯複用 | ✅ | ✅ | ✅ |
| 可組合性 | 中（巢狀包裹） | 中（回呼巢狀） | 高（直接呼叫） |
| TypeScript 支援 | 困難 | 中等 | 容易 |
| 除錯便利性 | 低（層層包裹） | 中 | 高 |
| 推薦程度 | 低 | 中 | 高 |

## Container / Presentational 模式

將元件分為邏輯層（Container）和展示層（Presentational）：

```jsx
// Presentational Component — 純展示，接收 props
function UserCard({ name, avatar, email, onFollow }) {
  return (
    <div className="user-card">
      <img src={avatar} alt={name} />
      <h3>{name}</h3>
      <p>{email}</p>
      <button onClick={onFollow}>追蹤</button>
    </div>
  );
}

// Container Component — 處理邏輯與資料
function UserCardContainer({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    fetchUser(userId).then(setUser);
  }, [userId]);

  const handleFollow = async () => {
    await followUser(userId);
  };

  if (!user) return <Skeleton />;

  return (
    <UserCard
      name={user.name}
      avatar={user.avatar}
      email={user.email}
      onFollow={handleFollow}
    />
  );
}
```

在 Hook 時代，這個模式演變為將邏輯抽到 Custom Hook 中：

```jsx
// 用 Custom Hook 取代 Container
function useUser(userId) {
  const [user, setUser] = useState(null);
  useEffect(() => { fetchUser(userId).then(setUser); }, [userId]);
  const handleFollow = () => followUser(userId);
  return { user, handleFollow };
}

function UserCard({ userId }) {
  const { user, handleFollow } = useUser(userId);
  if (!user) return <Skeleton />;

  return (
    <div className="user-card">
      <img src={user.avatar} alt={user.name} />
      <h3>{user.name}</h3>
      <button onClick={handleFollow}>追蹤</button>
    </div>
  );
}
```

## Provider 模式

利用 Context API 建立全域或區域的資料供應者：

```jsx
// 建立完整的 Theme Provider
const ThemeContext = createContext(null);

function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');

  const toggleTheme = () => {
    setTheme(prev => prev === 'light' ? 'dark' : 'light');
  };

  const value = useMemo(() => ({
    theme,
    toggleTheme,
    colors: theme === 'light'
      ? { bg: '#fff', text: '#000', primary: '#3b82f6' }
      : { bg: '#1a1a1a', text: '#fff', primary: '#60a5fa' }
  }), [theme]);

  return (
    <ThemeContext.Provider value={value}>
      {children}
    </ThemeContext.Provider>
  );
}

// 搭配 Custom Hook 使用
function useTheme() {
  const context = useContext(ThemeContext);
  if (!context) {
    throw new Error('useTheme 必須在 ThemeProvider 內使用');
  }
  return context;
}
```

### 多 Provider 組合

```jsx
// 避免 Provider 地獄
function AppProviders({ children }) {
  return (
    <AuthProvider>
      <ThemeProvider>
        <I18nProvider>
          <NotificationProvider>
            {children}
          </NotificationProvider>
        </I18nProvider>
      </ThemeProvider>
    </AuthProvider>
  );
}

// 或用 compose 函式簡化
function composeProviders(...providers) {
  return ({ children }) =>
    providers.reduceRight(
      (acc, Provider) => <Provider>{acc}</Provider>,
      children
    );
}

const AppProviders = composeProviders(
  AuthProvider, ThemeProvider, I18nProvider, NotificationProvider
);
```

## Controlled vs Uncontrolled 模式

這是 React 表單處理的兩種基本模式：

### Controlled Component

```jsx
function ControlledInput() {
  const [value, setValue] = useState('');

  const handleChange = (e) => {
    // 完全控制輸入值 — 可以加入驗證、格式化等
    const newValue = e.target.value.toUpperCase();
    setValue(newValue);
  };

  return (
    <input
      value={value}        // React 控制值
      onChange={handleChange} // 每次變更都經過 React
    />
  );
}
```

### Uncontrolled Component

```jsx
function UncontrolledInput() {
  const inputRef = useRef(null);

  const handleSubmit = (e) => {
    e.preventDefault();
    // 需要時才讀取 DOM 值
    console.log('輸入值：', inputRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        ref={inputRef}        // 透過 ref 存取
        defaultValue="初始值"  // 只設定初始值
      />
      <button type="submit">送出</button>
    </form>
  );
}
```

### 建立同時支援兩種模式的元件

```jsx
function Toggle({ value: controlledValue, defaultValue = false, onChange }) {
  const [internalValue, setInternalValue] = useState(defaultValue);

  // 判斷是否為受控模式
  const isControlled = controlledValue !== undefined;
  const value = isControlled ? controlledValue : internalValue;

  const handleToggle = () => {
    const newValue = !value;
    if (!isControlled) {
      setInternalValue(newValue);
    }
    onChange?.(newValue);
  };

  return (
    <button onClick={handleToggle}>
      {value ? 'ON' : 'OFF'}
    </button>
  );
}

// 受控使用
<Toggle value={isOn} onChange={setIsOn} />

// 非受控使用
<Toggle defaultValue={true} onChange={(v) => console.log(v)} />
```

## Slot 模式

Slot 模式讓父元件可以將內容插入子元件的特定位置：

```jsx
function Card({ header, footer, children }) {
  return (
    <div className="card">
      {header && <div className="card-header">{header}</div>}
      <div className="card-body">{children}</div>
      {footer && <div className="card-footer">{footer}</div>}
    </div>
  );
}

// 使用方式
<Card
  header={<h2>標題</h2>}
  footer={<button>確認</button>}
>
  <p>這是卡片內容</p>
</Card>
```

### 進階 Slot：使用 Children Utilities

```jsx
import { Children, isValidElement } from 'react';

function Dialog({ children }) {
  let title = null;
  let body = null;
  let actions = null;

  Children.forEach(children, child => {
    if (!isValidElement(child)) return;
    switch (child.type) {
      case DialogTitle:   title = child; break;
      case DialogBody:    body = child; break;
      case DialogActions: actions = child; break;
    }
  });

  return (
    <div className="dialog-overlay">
      <div className="dialog">
        {title && <div className="dialog-title">{title}</div>}
        {body && <div className="dialog-body">{body}</div>}
        {actions && <div className="dialog-actions">{actions}</div>}
      </div>
    </div>
  );
}

function DialogTitle({ children }) { return children; }
function DialogBody({ children }) { return children; }
function DialogActions({ children }) { return children; }

// 使用方式 — 順序不重要
<Dialog>
  <DialogActions>
    <button>取消</button>
    <button>確認</button>
  </DialogActions>
  <DialogTitle>確認刪除</DialogTitle>
  <DialogBody>確定要刪除這個項目嗎？</DialogBody>
</Dialog>
```

## 模式選擇指南

面對不同需求時，如何選擇合適的模式：

| 需求 | 推薦模式 |
|------|----------|
| 複用狀態邏輯 | Custom Hook |
| 元件組合（多子元件協同） | Compound Components |
| 渲染邏輯自訂（彈性渲染） | Render Props / Slot |
| 全域狀態管理 | Provider 模式 |
| 表單元件 | Controlled + Uncontrolled 雙模式 |
| 權限/認證包裹 | HOC（或 Custom Hook + 條件渲染） |
| 展示/邏輯分離 | Container/Presentational（或 Custom Hook） |

### 決策流程

```
需要複用邏輯嗎？
├── 是 → 優先使用 Custom Hook
│         需要包裹 UI 嗎？
│         ├── 是 → 考慮 HOC 或 Render Props
│         └── 否 → Custom Hook 即可
└── 否 → 需要多元件協同嗎？
          ├── 是 → Compound Components
          └── 否 → 需要全域共享嗎？
                    ├── 是 → Provider 模式
                    └── 否 → 一般 Props 傳遞即可
```

## 面試重點整理

1. **Compound Components**：透過 Context 隱式共享狀態，提供靈活的元件組合 API
2. **Render Props**：將渲染邏輯委託給使用者，透過函式 props 實現
3. **HOC**：接受元件回傳增強元件的高階函式，適合橫切關注點
4. **Custom Hook**：現代 React 邏輯複用的首選方式，取代大部分 HOC/Render Props 場景
5. **Controlled vs Uncontrolled**：受控元件由 React 管理值，非受控元件由 DOM 管理
6. **Provider 模式**：搭配 Context API 實現跨層級資料共享
7. **模式選擇**：沒有萬能模式，根據實際需求選擇最適合的方式
