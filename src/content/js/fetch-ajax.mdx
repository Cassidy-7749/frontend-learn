# Fetch API 與 AJAX

AJAX（Asynchronous JavaScript and XML）是一種在不重新載入頁面的情況下，與伺服器交換資料的技術。現代開發中，我們主要使用 Fetch API 來取代傳統的 XMLHttpRequest。

## XMLHttpRequest 簡介

XMLHttpRequest（XHR）是最早實現 AJAX 的方式，雖然現在已被 Fetch API 取代，但了解它有助於理解歷史脈絡：

```js
const xhr = new XMLHttpRequest();
xhr.open('GET', 'https://api.example.com/data');
xhr.onreadystatechange = function () {
  if (xhr.readyState === 4 && xhr.status === 200) {
    const data = JSON.parse(xhr.responseText);
    console.log(data);
  }
};
xhr.onerror = function () {
  console.error('請求失敗');
};
xhr.send();
```

<Callout type="info" title="readyState 狀態碼">
XHR 的 `readyState` 有五種狀態：0（UNSENT）、1（OPENED）、2（HEADERS_RECEIVED）、3（LOADING）、4（DONE）。只有狀態為 4 時表示請求完成。
</Callout>

## Fetch API 基本用法

Fetch API 是現代瀏覽器內建的網路請求介面，回傳 Promise，語法更直觀：

```js
// 基本 GET 請求
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('錯誤:', error));

// 使用 async/await
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('錯誤:', error);
  }
}
```

## Request 與 Response 物件

Fetch API 的核心圍繞著 `Request` 和 `Response` 兩個物件：

```js
// 建立 Request 物件
const request = new Request('https://api.example.com/data', {
  method: 'GET',
  headers: new Headers({ 'Accept': 'application/json' }),
  mode: 'cors',
  cache: 'default',
});

// Response 常用屬性
fetch(request).then(response => {
  console.log(response.ok);         // 狀態碼 200-299 為 true
  console.log(response.status);     // HTTP 狀態碼
  console.log(response.statusText); // 狀態文字
  console.log(response.headers);    // Response Headers
  console.log(response.url);        // 請求的 URL
});
```

Response body 只能被讀取一次，常用方法包括：

```js
response.json();       // 解析為 JSON
response.text();       // 解析為純文字
response.blob();       // 解析為 Blob（如圖片）
response.arrayBuffer(); // 解析為 ArrayBuffer
response.formData();   // 解析為 FormData
```

## Headers 操作

Headers 物件用於操作 HTTP 標頭：

```js
const headers = new Headers();
headers.append('Content-Type', 'application/json');
headers.append('Authorization', 'Bearer token123');

// 讀取
console.log(headers.get('Content-Type'));

// 檢查
console.log(headers.has('Authorization')); // true

// 刪除
headers.delete('Authorization');

// 遍歷
for (const [key, value] of headers) {
  console.log(`${key}: ${value}`);
}
```

## POST 請求與 JSON

發送 POST 請求並傳送 JSON 資料是最常見的操作之一：

```js
async function createUser(userData) {
  const response = await fetch('https://api.example.com/users', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(userData),
  });

  if (!response.ok) {
    throw new Error(`HTTP 錯誤: ${response.status}`);
  }

  return await response.json();
}

// 發送 FormData
async function uploadFile(file) {
  const formData = new FormData();
  formData.append('file', file);

  const response = await fetch('/upload', {
    method: 'POST',
    body: formData, // 不需手動設定 Content-Type
  });

  return await response.json();
}
```

## 錯誤處理（HTTP 狀態碼）

<Callout type="warning" title="Fetch 不會因 HTTP 錯誤拋出異常">
Fetch 只有在網路錯誤（如斷網）時才會 reject Promise。即使伺服器回傳 404 或 500，Promise 仍然會 resolve。你必須自行檢查 `response.ok` 或 `response.status`。
</Callout>

```js
async function safeFetch(url) {
  try {
    const response = await fetch(url);

    if (!response.ok) {
      // 根據狀態碼處理不同錯誤
      switch (response.status) {
        case 400: throw new Error('請求格式錯誤');
        case 401: throw new Error('未授權，請重新登入');
        case 403: throw new Error('沒有存取權限');
        case 404: throw new Error('資源不存在');
        case 500: throw new Error('伺服器內部錯誤');
        default:  throw new Error(`HTTP 錯誤: ${response.status}`);
      }
    }

    return await response.json();
  } catch (error) {
    if (error.name === 'TypeError') {
      console.error('網路連線失敗');
    }
    throw error;
  }
}
```

## AbortController

AbortController 用於取消正在進行的 Fetch 請求，常用於元件卸載或搜尋防抖：

```js
const controller = new AbortController();
const { signal } = controller;

// 設定超時自動取消
const timeoutId = setTimeout(() => controller.abort(), 5000);

try {
  const response = await fetch('/api/data', { signal });
  clearTimeout(timeoutId);
  const data = await response.json();
} catch (error) {
  if (error.name === 'AbortError') {
    console.log('請求已被取消');
  }
}

// 手動取消
controller.abort();
```

## 最佳實踐

1. **統一封裝**：建立通用的 fetch 函式，統一處理錯誤、token 注入、回應解析
2. **超時控制**：搭配 AbortController 設定請求超時
3. **錯誤分層**：區分網路錯誤與業務邏輯錯誤
4. **快取策略**：善用 `cache` 選項與 HTTP 快取標頭
5. **避免競態條件**：使用 AbortController 取消過期的請求
