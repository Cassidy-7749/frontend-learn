# TypeScript 與 React 實戰

React 與 TypeScript 的結合已成為現代前端開發的主流選擇。TypeScript 為 React 元件提供了完整的型別安全，從 Props 定義到 Hook 使用，大幅減少執行時期錯誤。本篇將深入介紹在 React 專案中如何善用 TypeScript 的各種實務技巧。

## FC 型別與函式元件

定義 React 函式元件有兩種常見方式：

```tsx
import { FC } from 'react';

// 方式一：使用 FC 型別（Function Component）
const Greeting: FC<{ name: string }> = ({ name }) => {
  return <h1>Hello, {name}</h1>;
};

// 方式二：直接標註參數型別（推薦）
function Greeting2({ name }: { name: string }) {
  return <h1>Hello, {name}</h1>;
}

// 或將 Props 抽出為獨立型別
interface GreetingProps {
  name: string;
}

const Greeting3 = ({ name }: GreetingProps) => {
  return <h1>Hello, {name}</h1>;
};
```

### FC 的爭議

`React.FC` 在早期版本會隱含 `children` 屬性，且有一些限制。React 18 已移除隱含的 `children`，但社群仍偏好直接標註型別：

```tsx
// FC 的特點
const MyComponent: FC<Props> = (props) => { /* ... */ return null; };

// FC 隱含的型別資訊
// - 回傳型別自動為 ReactElement | null
// - 包含 displayName、defaultProps 等靜態屬性

// 直接標註的優點
// - 更明確、更靈活
// - 支援泛型元件（FC 不方便做泛型）
// - 更好的型別推導
function MyComponent2(props: Props): JSX.Element {
  // ...
  return <div />;
}
```

<Callout type="info" title="推薦寫法">
目前社群主流推薦直接在函式參數上標註 Props 型別，而非使用 `React.FC`。這樣更靈活，支援泛型，且型別意圖更明確。但使用 `FC` 也完全沒問題，選擇團隊一致的風格即可。
</Callout>

## Props 型別定義：interface vs type

定義 Props 時，`interface` 和 `type` 都可以使用：

```tsx
// 使用 interface
interface ButtonProps {
  label: string;
  variant?: 'primary' | 'secondary' | 'danger';
  disabled?: boolean;
  onClick: () => void;
}

// 使用 type
type ButtonProps2 = {
  label: string;
  variant?: 'primary' | 'secondary' | 'danger';
  disabled?: boolean;
  onClick: () => void;
};

// 兩者在大多數情況下可互換
// interface 支援 declaration merging
// type 支援聯合型別、mapped types 等
```

### Props 進階模式

```tsx
// 條件 Props — 互斥屬性
type TextInputProps = {
  type: 'text';
  maxLength?: number;
};

type NumberInputProps = {
  type: 'number';
  min?: number;
  max?: number;
};

type InputProps = TextInputProps | NumberInputProps;

function Input(props: InputProps) {
  if (props.type === 'number') {
    // TypeScript 知道這裡有 min 和 max
    console.log(props.min, props.max);
  }
  return <input type={props.type} />;
}

// 展開 Props — 繼承 HTML 元素屬性
interface CustomButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary';
  isLoading?: boolean;
}

function CustomButton({ variant, isLoading, children, ...rest }: CustomButtonProps) {
  return (
    <button
      className={`btn btn-${variant}`}
      disabled={isLoading}
      {...rest}
    >
      {isLoading ? 'Loading...' : children}
    </button>
  );
}
```

## children 型別

React 中 `children` 的型別有幾種選擇：

```tsx
import { ReactNode, PropsWithChildren } from 'react';

// 方式一：明確定義 children
interface CardProps {
  title: string;
  children: ReactNode;  // 最常用，接受任何可渲染內容
}

function Card({ title, children }: CardProps) {
  return (
    <div className="card">
      <h2>{title}</h2>
      <div>{children}</div>
    </div>
  );
}

// 方式二：使用 PropsWithChildren
interface CardProps2 {
  title: string;
}

function Card2({ title, children }: PropsWithChildren<CardProps2>) {
  return (
    <div className="card">
      <h2>{title}</h2>
      <div>{children}</div>
    </div>
  );
}

// 方式三：限制 children 型別
interface ListProps {
  children: ReactNode;  // 任何可渲染內容
}

interface StrictListProps {
  children: string;  // 只接受字串
}

interface RenderProps {
  children: (data: { count: number }) => ReactNode;  // Render Props 模式
}
```

### ReactNode vs ReactElement vs JSX.Element

```tsx
// ReactNode — 最寬泛，包含所有可渲染型別
// string | number | boolean | null | undefined | ReactElement | ReactFragment | ReactPortal
type Example1 = ReactNode;

// ReactElement — React.createElement 的回傳型別
// 不包含 string、number 等原始型別
type Example2 = React.ReactElement;

// JSX.Element — 等同於 ReactElement<any, any>
// 是 JSX 表達式的回傳型別
type Example3 = JSX.Element;

// 建議：
// - children 用 ReactNode
// - 元件回傳值通常自動推導，不需手動標註
```

## 事件型別

React 事件有完整的型別系統：

```tsx
import { ChangeEvent, MouseEvent, FormEvent, KeyboardEvent } from 'react';

function EventExamples() {
  // 表單輸入事件
  const handleChange = (e: ChangeEvent<HTMLInputElement>) => {
    console.log(e.target.value);  // string，有完整自動補全
  };

  // 滑鼠事件
  const handleClick = (e: MouseEvent<HTMLButtonElement>) => {
    console.log(e.clientX, e.clientY);
  };

  // 表單提交
  const handleSubmit = (e: FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    const formData = new FormData(e.currentTarget);
  };

  // 鍵盤事件
  const handleKeyDown = (e: KeyboardEvent<HTMLInputElement>) => {
    if (e.key === 'Enter') {
      console.log('Enter pressed');
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <input onChange={handleChange} onKeyDown={handleKeyDown} />
      <button onClick={handleClick}>Submit</button>
    </form>
  );
}
```

### 事件處理器型別

```tsx
import { ChangeEventHandler, MouseEventHandler } from 'react';

// 可以直接使用 EventHandler 型別
const handleChange: ChangeEventHandler<HTMLInputElement> = (e) => {
  // e 的型別自動推導為 ChangeEvent<HTMLInputElement>
  console.log(e.target.value);
};

const handleClick: MouseEventHandler<HTMLButtonElement> = (e) => {
  console.log(e.button);
};

// 自訂事件處理器型別
type OnValueChange = (value: string, field: string) => void;

interface FormFieldProps {
  name: string;
  value: string;
  onChange: OnValueChange;
}
```

## useState / useRef / useReducer 泛型

### useState

```tsx
import { useState } from 'react';

// 自動推導
const [count, setCount] = useState(0);          // number
const [name, setName] = useState('');            // string

// 明確指定型別
const [user, setUser] = useState<User | null>(null);

interface User {
  id: number;
  name: string;
  email: string;
}

// 複雜狀態
interface FormState {
  values: Record<string, string>;
  errors: Record<string, string>;
  isSubmitting: boolean;
}

const [form, setForm] = useState<FormState>({
  values: {},
  errors: {},
  isSubmitting: false,
});
```

### useRef

```tsx
import { useRef } from 'react';

// DOM 元素引用 — 初始值為 null
const inputRef = useRef<HTMLInputElement>(null);
// inputRef.current 的型別是 HTMLInputElement | null

function focusInput() {
  inputRef.current?.focus();  // 需要可選鏈
}

// 可變值引用 — 初始值非 null
const countRef = useRef<number>(0);
// countRef.current 的型別是 number（可直接修改）

countRef.current += 1;

// 計時器引用
const timerRef = useRef<ReturnType<typeof setInterval> | null>(null);

function startTimer() {
  timerRef.current = setInterval(() => {
    console.log('tick');
  }, 1000);
}

function stopTimer() {
  if (timerRef.current) {
    clearInterval(timerRef.current);
    timerRef.current = null;
  }
}
```

### useReducer

```tsx
import { useReducer } from 'react';

// 定義 State 和 Action 型別
interface CounterState {
  count: number;
}

type CounterAction =
  | { type: 'increment' }
  | { type: 'decrement' }
  | { type: 'reset'; payload: number };

function counterReducer(state: CounterState, action: CounterAction): CounterState {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };
    case 'reset':
      return { count: action.payload };  // payload 有正確型別
    default:
      return state;
  }
}

// 使用
function Counter() {
  const [state, dispatch] = useReducer(counterReducer, { count: 0 });

  return (
    <div>
      <p>{state.count}</p>
      <button onClick={() => dispatch({ type: 'increment' })}>+</button>
      <button onClick={() => dispatch({ type: 'reset', payload: 0 })}>Reset</button>
    </div>
  );
}
```

<Callout type="tip" title="Discriminated Unions">
使用 `type` 欄位作為辨別器的聯合型別（Discriminated Unions）是定義 Action 型別的最佳模式。TypeScript 能在 `switch` 中自動收窄型別，確保每個 case 都能存取正確的 `payload`。
</Callout>

## Context 型別

```tsx
import { createContext, useContext, PropsWithChildren } from 'react';

// 定義 Context 型別
interface ThemeContextType {
  theme: 'light' | 'dark';
  toggleTheme: () => void;
}

// 建立 Context（使用 null 作為預設值）
const ThemeContext = createContext<ThemeContextType | null>(null);

// 自訂 Hook 加上型別守衛
function useTheme(): ThemeContextType {
  const context = useContext(ThemeContext);
  if (!context) {
    throw new Error('useTheme must be used within a ThemeProvider');
  }
  return context;  // 回傳型別已排除 null
}

// Provider 元件
function ThemeProvider({ children }: PropsWithChildren) {
  const [theme, setTheme] = useState<'light' | 'dark'>('light');

  const toggleTheme = () => {
    setTheme(prev => prev === 'light' ? 'dark' : 'light');
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

// 消費端使用
function Header() {
  const { theme, toggleTheme } = useTheme();  // 完整型別安全
  return <button onClick={toggleTheme}>Current: {theme}</button>;
}
```

## forwardRef 型別

```tsx
import { forwardRef, useImperativeHandle } from 'react';

// 基本 forwardRef
interface InputProps {
  label: string;
  error?: string;
}

const TextInput = forwardRef<HTMLInputElement, InputProps>(
  ({ label, error }, ref) => {
    return (
      <div>
        <label>{label}</label>
        <input ref={ref} />
        {error && <span className="error">{error}</span>}
      </div>
    );
  }
);

TextInput.displayName = 'TextInput';

// 使用 useImperativeHandle 暴露自訂方法
interface FormHandle {
  reset: () => void;
  validate: () => boolean;
  getValues: () => Record<string, string>;
}

interface FormProps {
  onSubmit: (values: Record<string, string>) => void;
}

const Form = forwardRef<FormHandle, PropsWithChildren<FormProps>>(
  ({ onSubmit, children }, ref) => {
    useImperativeHandle(ref, () => ({
      reset: () => { /* 重設表單 */ },
      validate: () => { /* 驗證 */ return true; },
      getValues: () => { /* 取得值 */ return {}; },
    }));

    return <form>{children}</form>;
  }
);

Form.displayName = 'Form';

// 消費端
function App() {
  const formRef = useRef<FormHandle>(null);

  const handleSave = () => {
    if (formRef.current?.validate()) {
      const values = formRef.current.getValues();
      console.log(values);
    }
  };

  return <Form ref={formRef} onSubmit={console.log}>...</Form>;
}
```

## 自訂 Hook 型別

```tsx

interface UseFetchResult<T> {

  isLoading: boolean;

  // ... fetch 邏輯

  return { data, error, isLoading, refetch };

// 元組回傳值（類似 useState）
function useToggle(initial = false): [boolean, () => void] {
  const [value, setValue] = useState(initial);
  const toggle = () => setValue(v => !v);
  return [value, toggle];  // 需要明確標註回傳型別為元組
}

// 使用 as const 也可以
function useToggle2(initial = false) {
  const [value, setValue] = useState(initial);
  const toggle = () => setValue(v => !v);
  return [value, toggle] as const;
  // readonly [boolean, () => void]
}

// 泛型 Hook
function useLocalStorage<T>(key: string, initialValue: T) {
  const [storedValue, setStoredValue] = useState<T>(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? (JSON.parse(item) as T) : initialValue;
    } catch {
      return initialValue;
    }
  });

  const setValue = (value: T | ((prev: T) => T)) => {
    const valueToStore = value instanceof Function ? value(storedValue) : value;
    setStoredValue(valueToStore);
    window.localStorage.setItem(key, JSON.stringify(valueToStore));
  };

  return [storedValue, setValue] as const;
}
```

## 常見型別工具

React 提供許多實用的型別工具：

```tsx
import {
  ComponentProps,
  ComponentPropsWithRef,
  ComponentPropsWithoutRef,
  ElementType,
} from 'react';

// ComponentProps — 提取元件的 Props 型別
type ButtonNativeProps = ComponentProps<'button'>;
type MyComponentProps = ComponentProps<typeof MyComponent>;

// 建立多型元件（Polymorphic Component）
interface PolymorphicProps<C extends ElementType> {
  as?: C;
  children?: ReactNode;
}

type Props<C extends ElementType> = PolymorphicProps<C> &
  Omit<ComponentPropsWithoutRef<C>, keyof PolymorphicProps<C>>;

function Box<C extends ElementType = 'div'>({ as, ...props }: Props<C>) {
  const Component = as || 'div';
  return <Component {...props} />;
}

// 使用
<Box as="a" href="/home">Link</Box>        // 有 <a> 的所有屬性
<Box as="button" onClick={() => {}}>Btn</Box> // 有 <button> 的所有屬性

// 其他實用型別
type HTMLProps = React.HTMLAttributes<HTMLDivElement>;
type SVGProps = React.SVGAttributes<SVGSVGElement>;
type CSSProps = React.CSSProperties;

const style: CSSProps = {
  display: 'flex',
  justifyContent: 'center',
  gap: '1rem',
};
```

<Callout type="warning" title="ComponentProps 的版本差異">
在 React 18+ 中，`ComponentProps` 會自動包含 `ref`。如果你需要明確排除或包含 `ref`，使用 `ComponentPropsWithoutRef` 或 `ComponentPropsWithRef`。
</Callout>

## 面試重點整理

1. **FC vs 直接標註** — 社群推薦直接在參數上標註 Props 型別，更靈活且支援泛型
2. **children 型別** — 使用 `ReactNode` 最為通用，`PropsWithChildren` 是快捷方式
3. **事件型別** — `ChangeEvent<HTMLInputElement>`、`MouseEvent<HTMLButtonElement>` 等，泛型參數指定 DOM 元素
4. **useState 泛型** — 當初始值為 `null` 時需要明確標註：`useState<Type | null>(null)`
5. **useRef 兩種模式** — `useRef<HTMLElement>(null)` 用於 DOM 引用；`useRef<T>(value)` 用於可變值
6. **useReducer Action** — 使用 Discriminated Unions 定義 Action，確保每個 case 的 payload 型別正確
7. **Context 非 null 模式** — 建立 `createContext<T | null>(null)` 搭配自訂 Hook 拋出錯誤
8. **泛型元件** — 不使用 FC，直接 `function List<T>(props: ListProps<T>)` 實現泛型
9. **forwardRef** — 第一個泛型參數是 ref 型別，第二個是 Props 型別
10. **ComponentProps** — 用來提取現有元件或 HTML 元素的 Props 型別
